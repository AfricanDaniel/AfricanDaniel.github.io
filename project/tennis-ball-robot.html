Tennis Ball Retrieval Robot

ROS2PythonC++YOLOv8OpenCVZED 2iRealSenseJetson Orin NanoLinux

Overview

An autonomous robot built to retrieve tennis balls on a court and return them to a designated drop zone. The system uses a mecanum-wheel chassis for omnidirectional movement, a servo-controlled claw for ball pickup, and a dual-camera perception stack split across a Jetson Orin Nano (vision) and a Raspberry Pi (motion control) communicating over ROS2.

Hardware

  • Mecanum-wheel chassis driven by 4 brushless motors via ODrive ESCs on CAN bus
  • ZED 2i stereo camera for far-range 3D ball detection and visual odometry
  • Intel RealSense D435i RGBD camera for close-range detection and blind-spot handling
  • 2-DOF servo arm (Dynamixel) with a claw for grasping and releasing tennis balls
  • Jetson Orin Nano running the perception stack; Raspberry Pi running motion control

Perception Pipeline

A custom YOLOv8 model (trained on court footage) runs on the Jetson and detects three classes simultaneously: tennis ball, drop-zone water bottle, and court net strap. The ZED 2i handles far-range detection using the ZED SDK's built-in AI object detection. The RealSense takes over at close range, with a fallback that estimates z=0.10m when depth returns NaN due to the camera blind spot.

Autonomous Mission

State Machine (ball_chaser.py)

  1. 1INIT: Lock the drop-zone bottle position via TF, rotate to face the court.
  2. 2WAITING: Monitor ball stillness — chase only begins after the ball is stationary for 3 seconds.
  3. 3CHASING (ZED): Proportional angular + fixed linear drive toward ball using ZED far-range detection.
  4. 4CHASING (RealSense): Handoff to close-range camera; arm tilts down as distance decreases.
  5. 5GRABBING: Arm sweeps down, claw closes, arm raises.
  6. 6RETURN: Navigate back to bottle drop zone using odometry and visual re-acquisition.
  7. 7DROPPING: Arm sweeps down, claw opens, arm raises. Repeat mission.

Key Technical Challenges

  • Camera handoff: Seamless transition between ZED (far) and RealSense (close) to handle each camera's blind spot.
  • Ball stillness detection: Prevents chasing rolling balls by requiring 3s of positional stability within a 0.1m × 0.4m window.
  • Drop zone exclusion: Logs dropped ball positions to avoid re-chasing just-released balls.
  • ODrive initialization: Robust retry logic ensures all 4 motor axes enter closed-loop control before accepting commands.
  • Distributed compute: Vision runs on Jetson GPU, control runs on Pi — coordinated purely through ROS2 topics over the network.
